ABOUT FLOW PLANNER

The Challenge

Imagine you're building part of an AI multi-agent system. Agent A sends your agent, Agent B, different questions at runtime:

  • "How do I create a project in Linear?"
  • "How do I filter a database in Notion?"
  • "How do I add a task in Asana?"

Your goal: Build a system that shows Agent A how to perform the requested task. It should automatically navigate the live app and capture screenshots of each UI state in the workflow.

Agent B won't know these tasks ahead of time, so your system must be generalizable: it should handle any kind of request, across different web apps, and capture the right UI states in real time.

The Problem

Not every UI state has a URL. Consider what needs to be captured for "creating a project in Linear":

  • The project list page (has a URL)
  • The "Create Project" button state
  • The create modal (no URL)
  • The form fields (no URL)
  • Maybe the success state

Your system needs to navigate the live application and capture these states programmatically on the fly.

The Solution: Flow Planner

This project demonstrates a working solution using a multi-agent architecture:

Architecture

  • Gemini Planner: Takes natural language tasks and creates a high-level workflow plan
  • Browser-Use Agent: Autonomous browser agent powered by Claude Sonnet that executes the workflow using vision + DOM understanding
  • State Capturer: Captures UI states with screenshots at each step, even for modal dialogs without URLs
  • Vision Refinement: Uses Claude Sonnet with vision to enhance guides with detailed UI element identification

Technologies

  • Backend: Flask + SocketIO for real-time updates
  • Browser Automation: Browser-Use (Playwright-based autonomous agent)
  • AI Models: Google Gemini (planning), Anthropic Claude Sonnet (execution & vision)
  • Frontend: Vanilla JavaScript with retro-styled UI

How It Works

  1. Plan: Gemini analyzes the task and creates a workflow outline
  2. Execute: Browser-Use agent autonomously navigates the app and performs actions
  3. Capture: Screenshots and state data are captured at each step
  4. Generate: AI creates a markdown guide with annotated screenshots
  5. Refine: Vision AI enhances the guide with detailed UI analysis
